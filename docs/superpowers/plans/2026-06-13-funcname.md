```rust
/// Rebuild the dynamic `FUNCNAME` array from the live function call stack.
/// `FUNCNAME[0]` is the currently-executing function, `[1]` its caller, etc.
/// (the reverse of `function_arg0`'s push order). When the stack is empty
/// (top level) `FUNCNAME` is unset, matching bash. Called by `call_function`
/// after every push/pop of `function_arg0`.
pub(crate) fn sync_funcname(&mut self) {
    if self.function_arg0.is_empty() {
        self.vars.remove("FUNCNAME");
        return;
    }
    let n = self.function_arg0.len();
    let elements: BTreeMap<usize, String> = (0..n)
        .map(|k| (k, self.function_arg0[n - 1 - k].clone()))
        .collect();
    self.vars.insert(
        "FUNCNAME".to_string(),
        Variable {
            value: VarValue::Indexed(elements),
            exported: false,
            readonly: false,
            integer: false,
        },
    );
}
```

```bash
#!/usr/bin/env bash
# Byte-identical bash<->huck harness for v151: FUNCNAME inside function bodies.
```

## Notes for the implementer
- **The whole feature is the two `sync_funcname()` calls + the method.** Reads need NO new code: `$FUNCNAME` resolves via `lookup_var`'s `scalar_view()` (element `[0]`); `${FUNCNAME[@]}`/`[i]`/`${#…}`/`${!…}`/`[*]` via the existing `get_array`-backed array expansion.
- **Reversal:** `function_arg0` is `[outer, …, inner]` (push order); `FUNCNAME[0]` must be the innermost/current, so `FUNCNAME[k] = function_arg0[len-1-k]`.
- **Single-exit `call_function`:** the pop at line 2947 runs on all paths, so the two sync calls suffice; do NOT scatter extra calls.
- **Do not** special-case `FUNCNAME` anywhere in `expand.rs` / `param_expansion.rs` / `lookup_var` — the stored-array approach makes that unnecessary.
