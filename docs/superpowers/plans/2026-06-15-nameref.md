**Tech Stack:** Rust, the huck shell. Spec: `docs/superpowers/specs/2026-06-15-nameref-design.md` (read its verified-bash-behavior list — it is the behavior contract).

And the resolution type + helper (add `ResolvedName` near `AssignDest`):

```rust
/// Result of following a nameref chain to its effective destination.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ResolvedName {
    /// A plain variable target (the original name if `name` is not a nameref).
    Name(String),
    /// An element target `base[subscript-text]` — the subscript is evaluated
    /// at the use site in the base's current shape.
    Element { name: String, subscript: String },
    /// A nameref whose target value is empty (attribute set, not yet bound).
    /// Carries the nameref's own name (assignment BINDS it; reads are unset).
    Unbound(String),
    /// A cycle was detected; the `circular name reference` warning was emitted.
    Cycle,
}
```

```rust
    /// Follows the nameref chain starting at `name` to its effective storage
    /// destination. A no-op for ordinary variables (returns `Name(name)`), so
    /// callers may resolve unconditionally. Detects cycles via a visited set
    /// and emits bash's `circular name reference` warning.
    pub fn resolve_nameref(&self, name: &str) -> ResolvedName {
        let mut current = name.to_string();
        let mut visited = std::collections::HashSet::new();
        loop {
            if !visited.insert(current.clone()) {
                eprintln!("huck: warning: {name}: circular name reference");
                return ResolvedName::Cycle;
            }
            match self.vars.get(&current) {
                Some(v) if v.nameref => {
                    let target = v.value.scalar_view().to_string();
                    if target.is_empty() {
                        return ResolvedName::Unbound(current);
                    }
                    // Element target `base[sub]`?
                    match crate::builtins::parse_subscripted_arg(&target) {
                        Ok(Some((base, sub))) => {
                            return ResolvedName::Element {
                                name: base.to_string(),
                                subscript: sub.to_string(),
                            };
                        }
                        _ => { current = target; } // plain name → follow the chain
                    }
                }
                _ => return ResolvedName::Name(current),
            }
        }
    }
```

NOTE: `parse_subscripted_arg` is currently in `src/builtins.rs`. If it is not `pub(crate)`, make it so (or move it to a shared module / replicate a tiny `split_subscript` helper in shell_state.rs that finds the first `[` and trailing `]`). Match its exact splitting semantics (`arr[0]` → `("arr","0")`, plain `x` → `None`).

```rust
    /// Reads `arr[subscript-text]` as a scalar, evaluating the subscript in
    /// arr's current shape (associative → string key; else → arith index).
    /// Used by nameref element-target resolution.
    fn lookup_nameref_element(&self, arr: &str, subscript: &str) -> Option<String> {
        let sub_word = crate::lexer::Word(vec![crate::lexer::WordPart::Literal {
            text: subscript.to_string(), quoted: false,
        }]);
        if self.get_associative(arr).is_some() {
            let key = crate::expand::eval_subscript_key(&sub_word, self);
            self.lookup_associative_element(arr, &key)
        } else {
            // read_only_arith path (no &mut): reuse the same evaluator
            // `is_set`/`-v` uses for subscripts. If eval fails, treat as index 0? No —
            // return None. Use crate::expand::eval_subscript with a read-only shim.
            self.read_only_eval_index(&sub_word, arr).and_then(|i| self.lookup_array_element(arr, i))
        }
    }
```

- [ ] **Step 1: Change the seam to resolve through namerefs**

The v159 `resolve_assign_target(&self, dest) -> AssignDest` (1224) is the single write chokepoint. Make it `&mut self`-free if possible (it needs only reads + `resolve_nameref` which is `&self`; element-subscript eval for the write path can use the read-only index evaluator, OR — since `assign()` has `&mut self` — make `resolve_assign_target` `&mut self`). Recommended signature: `fn resolve_assign_target(&mut self, dest: AssignDest) -> Option<AssignDest>`.

```rust
    fn resolve_assign_target(&mut self, dest: AssignDest) -> Option<AssignDest> {
        // Only Whole targets can be namerefs; an explicit Element dest
        // (a[i]=) is already concrete (the executor resolved its subscript),
        // but its NAME could still be a nameref to another array — resolve it.
        let name = dest.name().to_string();
        if !self.is_nameref(&name) {
            return Some(dest); // fast path: not a nameref, unchanged
        }
        match self.resolve_nameref(&name) {
            ResolvedName::Name(n) => Some(match dest {
                AssignDest::Whole(_) => AssignDest::Whole(n),
                AssignDest::Element { sub, .. } => AssignDest::Element { name: n, sub },
            }),
            ResolvedName::Element { name: arr, subscript } => {
                // Element target like `arr[0]`: evaluate the subscript now.
                let sub = self.eval_nameref_subscript(&arr, &subscript);
                Some(AssignDest::Element { name: arr, sub })
            }
            ResolvedName::Unbound(refname) => {
                // Assigning to an UNBOUND nameref BINDS it: store the value as
                // the nameref's own target. Return its own name so assign()
                // stores into it.
                Some(AssignDest::Whole(refname))
            }
            ResolvedName::Cycle => None, // warning already emitted; drop the write
        }
    }

    /// Evaluates a nameref element-target subscript into the funnel's Subscript
    /// (associative key vs indexed index), mirroring apply_one_assignment.
    fn eval_nameref_subscript(&mut self, arr: &str, subscript: &str) -> Subscript {
        let sub_word = crate::lexer::Word(vec![crate::lexer::WordPart::Literal {
            text: subscript.to_string(), quoted: false,
        }]);
        if self.get_associative(arr).is_some() {
            Subscript::Key(crate::expand::eval_subscript_key(&sub_word, self))
        } else {
            match crate::expand::eval_subscript(&sub_word, self, arr) {
                Ok(i) => Subscript::Index(i),
                Err(_) => Subscript::Index(0),
            }
        }
    }
```

NOTE on bind: returning `Whole(refname)` for `Unbound` makes `assign()` store the value into the nameref variable itself — which IS the bind (the nameref's value becomes the target name). Verify `declare -n r; r=x; declare -p r` → `declare -n r="x"`. One caveat: `assign()` applies the target's attributes; a plain nameref has none, so the raw value is stored. If `assign()`'s readonly/attribute logic would interfere with an unbound nameref bind, special-case it (a nameref being bound stores its raw value); but in practice a fresh nameref has no other attributes.

## Notes for the implementer

- **Additive invariant:** `resolve_nameref` on a non-nameref returns `Name(name)`; every hook fast-paths `!is_nameref` → unchanged. If ANY existing harness diffs, a hook is resolving when it shouldn't (likely you hooked a low-level reader the storage primitives call — resolve in the EXPANSION/lookup layer, never inside the storage primitives the funnel uses).
- **Borrow discipline:** `resolve_nameref`/`lookup_var`/`is_set` are `&self`; element-subscript evaluation on read paths must use the read-only index evaluator (the one `[[ -v name[sub] ]]` uses). The write seam may be `&mut self`.
- **Binding via raw `set()`:** declaration (`declare -n r=target`) and unbound-bind both store the target name via the RAW `shell.set()` so it is NOT dereferenced. Never use `try_set`/the funnel to store a nameref's target (post-Task-4 that would deref).
- **Deferred (log after merge as low divergences):** `for r in …` through a nameref (bash itself is inconsistent); `[[ -R name ]]` nameref test operator; element-target whose base is itself a nameref (deep element chains).
