# FUNCNEST enforcement + recursion backstop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enforce bash's `FUNCNEST` function-nesting limit and add a defensive internal backstop, so deep recursion yields a clean `maximum function nesting level exceeded` error (rc 1) instead of a Rust SIGABRT. This is `func`'s last blocker — it flips `func` to PASS (suite 8→9).

**Architecture:** A `Shell::funcnest_limit()` parser in `shell_state.rs` and a guard at the top of `call_function` in `executor.rs` that refuses a call exceeding the effective limit (the user's FUNCNEST, clamped to a hard backstop) before any frame/body/trap work.

**Tech Stack:** Rust (`huck-engine`); bash diff harness under `tests/scripts/`.

## Global Constraints

- Commit trailer on EVERY commit, verbatim: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.
- Run the FULL suite with `cargo test --workspace` (~3698 baseline) — plain `cargo test` skips most crates.
- Byte-faithfulness oracle is system `bash` (5.2.21).
- `cargo build --release --bin huck` is slow — long timeout (~480000ms).
- bash rules (verified): `FUNCNEST=N` (positive int) allows N nested calls and refuses the one that would reach depth N+1, emitting `<source>: line L: NAME: maximum function nesting level exceeded (N)` to stderr and returning rc 1 (which propagates); unset/`0`/negative/non-numeric = unlimited. bash ALSO segfaults on infinite recursion when FUNCNEST is unset.
- **CRITICAL — do NOT run deep recursion (near the 2048 backstop) inside `cargo test`:** debug builds on the smaller test-thread stack overflow well before 2048 and would ABORT the test runner. Unit tests use a SHALLOW `FUNCNEST=3`; the backstop value is validated only via the release-binary harness.

```rust
    /// Parse $FUNCNEST. Some(n) for a positive integer limit; None (unlimited)
    /// for unset / 0 / negative / non-numeric — matching bash.
    pub fn funcnest_limit(&self) -> Option<usize> {
        self.lookup_var("FUNCNEST")
            .and_then(|v| v.trim().parse::<i64>().ok())
            .filter(|&n| n > 0)
            .map(|n| n as usize)
    }
```

- [ ] **Step 7: Add the guard to `call_function`**

In `crates/huck-engine/src/executor.rs`, at the VERY TOP of `call_function` (before `let saved = std::mem::take(&mut shell.positional_args);` ~line 3660):

```rust
    // FUNCNEST enforcement + recursion backstop. Refuse a call that would exceed
    // the effective nesting limit BEFORE any frame/positional/local setup, so the
    // caller's statement simply sees rc 1 (matching bash). The backstop
    // (FUNCNEST_HARD_MAX) converts otherwise-unbounded recursion into a clean
    // error instead of a Rust stack-overflow abort.
    const FUNCNEST_HARD_MAX: usize = 2048;
    let limit = shell
        .funcnest_limit()
        .map_or(FUNCNEST_HARD_MAX, |n| n.min(FUNCNEST_HARD_MAX));
    let depth = shell
        .call_stack
        .iter()
        .filter(|f| matches!(f.kind, crate::shell_state::FrameKind::Function))
        .count();
    if depth >= limit {
        let prefix = shell.error_prefix(Some(name));
        {
            let mut err = err_writer(err_sink, sink);
            e!(&mut *err, "{prefix}maximum function nesting level exceeded ({limit})");
        }
        return ExecOutcome::Continue(1);
    }
```

```bash
# Backstop: unbounded recursion with no FUNCNEST must yield a clean error + rc 1,
# NOT a SIGABRT (134). bash segfaults here, so this is huck-only. Capture rc
# directly (no pipe — a pipe subshell would lose the FAIL increment).
bs=$(mktemp -d); printf 'f(){ f; }\nf\n' > "$bs/frag.sh"
( cd "$bs" && "$HUCK_BIN" ./frag.sh >/dev/null 2>err.txt ); rc=$?
if [[ "$rc" == 1 ]] && grep -q "maximum function nesting level exceeded" "$bs/err.txt"; then
  echo "PASS: backstop (clean error, no SIGABRT)"; PASS=$((PASS+1))
else
  echo "FAIL: backstop rc=$rc (expected 1, no abort)"; FAIL=$((FAIL+1))
fi
rm -rf "$bs"
```

## Notes for the implementer

- The guard MUST be the first thing in `call_function` — before the
  positional-args/getopts/local-scope setup — so a refused call mutates nothing.
- `depth >= limit` is the exact boundary: `FUNCNEST=3` refuses at depth 3 (the
  4th call), matching bash. The message shows the effective `limit` (the FUNCNEST
  value when set ≤ cap; the cap otherwise).
- NEVER run a ~2048-deep recursion under `cargo test` (debug + small test-thread
  stack overflows first). Unit tests stay at `FUNCNEST=3`; the cap is validated
  only by the release-binary harness backstop assertion.
- Do NOT raise the native stack or add a big-stack thread (out of scope).
- Headline success: `func` flips to PASS; cprint + herestr stay PASS.
