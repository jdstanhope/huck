# v179: `|&` pipe stdout+stderr Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Support `cmd1 |& cmd2` (bash shorthand for `cmd1 2>&1 | cmd2` — merge the producer's stderr into the pipe), resolving divergence M-51.

**Architecture:** Lexer-level desugar — on `|&`, emit the exact token sequence for `2>&1` followed by `Pipe`. The existing parser (incl. v176 compound-stage redirects) and executor handle `<left> 2>&1 | <right>` unchanged; no new AST/operator/parser/executor.

**Tech Stack:** Rust (`src/lexer.rs`); bash-diff harness.

**Spec:** `docs/superpowers/specs/2026-06-17-pipe-both-design.md`

**Branch:** `v179-pipe-both`

**Background facts (verified):**
- huck already parses + executes the desugar target `cmd 2>&1 | cmd2` correctly (Simple AND compound producers), matching bash.
- `2>&1` lexes to exactly: `Token::RedirFd(RedirFd::Number(2))`, `Token::Op(Operator::DupOut)`, `Token::Word("1")`.
- The lexer `'|'` arm is at `src/lexer.rs:633`; it flushes the pending left word, then dispatches `||`→`Or` / `|`→`Pipe`, with one `push_pos!(c_off, c_line)` after.
- `RedirFd`, `Operator::DupOut`, `Word`, `WordPart::Literal`, and `push_pos!` are all in scope in `lexer.rs`.
- M-51 is a **Missing features (Tier 2)** entry; on resolution delete it and set the Tier-2 count 18 → 17.

- [ ] **Step 1: Add the `|&` branch in the `'|'` arm**

The current dispatch (after the left-word flush) is:
```rust
                if chars.peek() == Some(&'|') {
                    chars.next();
                    tokens.push(Token::Op(Operator::Or));
                } else {
                    tokens.push(Token::Op(Operator::Pipe));
                }
                push_pos!(c_off, c_line);
```
Replace it with (move `push_pos!` into each branch so every pushed token gets a position):
```rust
                if chars.peek() == Some(&'|') {
                    chars.next();
                    tokens.push(Token::Op(Operator::Or));
                    push_pos!(c_off, c_line);
                } else if chars.peek() == Some(&'&') {
                    // `|&` is bash shorthand for `2>&1 |`: merge the left command's
                    // stderr into the pipe, then pipe. Desugar at the token level so
                    // the existing pipeline/redirect machinery (incl. v176
                    // compound-stage redirects) handles it unchanged.
                    chars.next(); // consume the '&' of `|&`
                    tokens.push(Token::RedirFd(crate::command::RedirFd::Number(2)));
                    push_pos!(c_off, c_line);
                    tokens.push(Token::Op(Operator::DupOut));
                    push_pos!(c_off, c_line);
                    tokens.push(Token::Word(Word(vec![WordPart::Literal {
                        text: "1".to_string(),
                        quoted: false,
                    }])));
                    push_pos!(c_off, c_line);
                    tokens.push(Token::Op(Operator::Pipe));
                    push_pos!(c_off, c_line);
                } else {
                    tokens.push(Token::Op(Operator::Pipe));
                    push_pos!(c_off, c_line);
                }
```
(Delete the trailing standalone `push_pos!(c_off, c_line);` that previously followed the if/else — it is now inside each branch.)

### Task 2: Bash-diff harness for `|&`

**Files:** Create `tests/scripts/pipe_both_diff_check.sh`.

- [ ] **Step 1: Write the harness**

Create `tests/scripts/pipe_both_diff_check.sh` with exactly:
```bash
#!/usr/bin/env bash
# Byte-identical bash<->huck harness for v179: `|&` (pipe stdout+stderr), bash
# shorthand for `2>&1 |`. Each case EXECUTES and asserts identical stdout+exit
# (the merged stderr flows through the pipe to the consumer's stdout).
set -u
HUCK_BIN="${HUCK_BIN:-$(pwd)/target/debug/huck}"
[[ -x "$HUCK_BIN" ]] || { echo "build huck first: $HUCK_BIN" >&2; exit 1; }
PASS=0; FAIL=0
check() {
    local label="$1" frag="$2" b bo h ho
    bo=$(bash --norc --noprofile -c "$frag" 2>/dev/null); b=$?
    ho=$("$HUCK_BIN" -c "$frag" 2>/dev/null); h=$?
    if [[ "$bo" == "$ho" && "$b" == "$h" ]]; then printf 'PASS: %s\n' "$label"; PASS=$((PASS+1))
    else printf 'FAIL: %s  bash(rc=%s)=[%s]  huck(rc=%s)=[%s]\n' "$label" "$b" "$bo" "$h" "$ho"; FAIL=$((FAIL+1)); fi
}

check "simple both streams"   'sh -c "echo O; echo E 1>&2" |& cat'
check "stderr reaches grep"   'sh -c "echo keep; echo DROPME 1>&2" |& grep keep'
check "only stderr"           'sh -c "echo ERR 1>&2" |& cat'
check "compound group"        '{ echo o; echo e 1>&2; } |& cat'
check "subshell producer"     '( echo o; echo e 1>&2 ) |& cat'
check "chained pipe-both"     'sh -c "echo x; echo y 1>&2" |& tr a-z A-Z |& cat'
check "pipe-both then sort"   'sh -c "echo b; echo a 1>&2" |& sort'
check "no-space form"         'sh -c "echo O;echo E 1>&2"|&cat'
check "control plain pipe"    'echo hi | tr a-z A-Z'
check "control logical or"    'false || echo alt'

echo ""; echo "Total: $((PASS+FAIL)), Pass: $PASS, Fail: $FAIL"
exit $(( FAIL > 0 ? 1 : 0 ))
```
