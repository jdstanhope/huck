# extglob String Matching Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Support extended glob patterns (`?(…)` `*(…)` `+(…)` `@(…)` `!(…)`, with `|`-alternation and nesting), gated on `shopt -s extglob`, in huck's three string-matching contexts: `[[ == / != ]]`, `case`, and parameter expansion (`${v#…}`/`##`/`%`/`%%`/`/`/`//`/`^`/`,`).

**Architecture:** A new pure matcher module `src/glob_match.rs` (`extglob_match`, `has_extglob`) does extglob matching by recursive backtracking; it is invoked only when extglob is on AND the pattern contains an extglob operator, so plain `*`/`?`/`[…]` patterns keep using the `glob` crate unchanged. The lexer gains `tokenize_with_opts(LexerOptions{extglob})` (with a back-compat `tokenize` wrapper) so `+(a|b)` lexes as one word when extglob is on — making `[[`/`case` parse it with no parser change. The three match sites dispatch to the engine.

**Tech Stack:** Rust; huck's `Token`/`Word`/`Shell`/`ShellOptions` (the v86 `extglob` shopt flag); the `glob` crate stays for non-extglob patterns.

**Spec:** `docs/superpowers/specs/2026-06-04-extglob-design.md`

**Verified bash 5.2 semantics (the engine's contract):**
- `?(L)`=0-or-1, `*(L)`=0+, `+(L)`=1+, `@(L)`=exactly one, `!(L)`=any span that does NOT match L. `L` is `|`-separated alts; alts contain `*`/`?`/`[…]`/nested groups.
- `?(abc)` matches `""`,`abc`, not `abcabc`; `+(ab)` matches `abab`, not `""`; `@(ab|cd)` matches `ab`,`cd`, not `abcd`; `!(bar)` matches `foo`, not `bar`; `a@(x|y)b`→`axb`; `a+(x|y)b`→`axxb`; `+([a-z]).txt`→`file.txt`; `@(a*(b)c)`→`abc`.
- `${v##+(a)}` (v=aaab)→`b`; `${v%%+(bar)}` (v=foobarbar)→`foo`; `${v/+(abc)/X}` (v=abcabc)→`X`.
- **Uniform flag-gating**: huck honors extglob (parse + match) only with `shopt -s extglob`. Documented divergence: bash honors extglob in `[[` even flag-off; huck requires the flag (real flag-on usage is fully faithful).

```rust
//! Extended-glob (`shopt extglob`) pattern matcher. Pure: no shell, no FS.
//! Used by `[[`/`case`/`${}` only when extglob is on AND the pattern contains
//! an extglob operator (`has_extglob`). Plain globs keep using the `glob` crate.

#[derive(Debug, Clone, Copy, PartialEq)]
enum GroupKind { ZeroOrOne, ZeroOrMore, OneOrMore, ExactlyOne, Not }

#[derive(Debug, Clone)]
enum Item {
    Lit(char),
    AnyChar,                                   // ?
    AnyRun,                                    // *
    Class { negated: bool, set: Vec<ClassAtom> },  // [...]
    Group { kind: GroupKind, alts: Vec<Vec<Item>> }, // ?( *( +( @( !(
}

#[derive(Debug, Clone)]
enum ClassAtom { Ch(char), Range(char, char) }

/// True if `pattern` contains an extglob operator: one of `? * + @ !` directly
/// followed by `(` (scanning past `\`-escapes and bracket classes).
pub fn has_extglob(pattern: &str) -> bool {
    let b: Vec<char> = pattern.chars().collect();
    let mut i = 0;
    while i < b.len() {
        match b[i] {
            '\\' => { i += 2; continue; }
            '?' | '*' | '+' | '@' | '!' if i + 1 < b.len() && b[i + 1] == '(' => return true,
            _ => i += 1,
        }
    }
    false
}

/// Matches `text` against extglob `pattern` (the WHOLE text must match).
pub fn extglob_match(pattern: &str, text: &str, case_insensitive: bool) -> bool {
    let pat = parse_seq(&pattern.chars().collect::<Vec<_>>(), &mut 0, false);
    let txt: Vec<char> = text.chars().collect();
    match_here(&pat, &txt, case_insensitive)
}
```

Implement these helpers (write them to pass the Step-1 tests — the tests are the contract):
- `parse_seq(chars, pos, in_group) -> Vec<Item>`: walk chars from `*pos`. `\` → `Lit(next)`. `[` → parse a `Class` (ranges via `a-z`, leading `!`/`^` → negated, literal `]` if first). `?`/`*`/`+`/`@`/`!` followed by `(` → parse a `Group` (consume `(`, then repeatedly `parse_seq(in_group=true)` splitting on top-level `|`, until the matching `)`); track nested paren depth. `?`→`AnyChar`, `*`→`AnyRun` (when NOT followed by `(`). Other char → `Lit`. When `in_group`, stop at a top-level `|` or `)` (return to caller to handle the split / close).
- `match_here(items, text, ci) -> bool` (anchored, whole-text):
  - `[]` empty items → `text.is_empty()`.
  - `Lit(c)` → `text` nonempty, `eqc(text[0],c,ci)`, recurse on tail.
  - `AnyChar` → `text` nonempty, recurse on tail.
  - `AnyRun` → `(0..=text.len()).any(|k| match_here(rest, &text[k..], ci))`.
  - `Class` → `text` nonempty, `class_matches(set, negated, text[0], ci)`, recurse tail.
  - `Group{kind, alts}` → dispatch:
    - `ExactlyOne`: `(0..=text.len()).any(|k| alt_matches_whole(alts, &text[..k], ci) && match_here(rest, &text[k..], ci))`.
    - `ZeroOrOne`: `match_here(rest, text, ci) || (1..=text.len()).any(|k| alt_matches_whole(alts,&text[..k],ci) && match_here(rest,&text[k..],ci))`.
    - `ZeroOrMore`: `match_star(alts, rest, text, ci)` where `match_star` = `match_here(rest,text,ci) || (1..=text.len()).any(|k| alt_matches_whole(alts,&text[..k],ci) && match_star(alts,rest,&text[k..],ci))`.
    - `OneOrMore`: `(1..=text.len()).any(|k| alt_matches_whole(alts,&text[..k],ci) && match_star(alts, rest, &text[k..], ci))`.
    - `Not`: `(0..=text.len()).any(|k| !alt_matches_whole(alts,&text[..k],ci) && match_here(rest,&text[k..],ci))`.
  where `alt_matches_whole(alts, span, ci) = alts.iter().any(|a| match_here(a, span, ci))`, `eqc(a,b,ci)` compares chars (lowercased if `ci`), and `class_matches` checks the bracket set (with `ci`). Slices are `&[char]`.

> Recursion is exponential in the worst case but bounded and fine for shell-length patterns. Keep `match_here` taking `&[Item]` and `&[char]` slices.

/// Pattern match for parameter expansion: extglob engine when enabled+applicable,
/// else the `glob` crate (preserving current behavior). `case_sensitive` mirrors

fn pe_pattern_matches(pattern: &str, text: &str, extglob: bool, case_sensitive: bool) -> bool {

}

```rust
/// Reads `X(...)` where X is the just-seen extglob prefix char, consuming a
/// balanced-paren group (nested parens; inner `|`/metachars literal). `chars`
/// is positioned just after the prefix char; returns the full group text
/// INCLUDING the prefix char, e.g. "+(a|b)".
fn scan_extglob_group(prefix: char, chars: &mut Peekable<Chars>) -> Result<String, LexError> {
    let mut out = String::new();
    out.push(prefix);
    out.push('('); chars.next(); // consume '('
    let mut depth = 1usize;
    while let Some(&c) = chars.peek() {
        chars.next();
        out.push(c);
        match c {
            '(' => depth += 1,
            ')' => { depth -= 1; if depth == 0 { return Ok(out); } }
            _ => {}
        }
    }
    Err(LexError::UnterminatedExtglob) // EOF before closing ')'
}
```
